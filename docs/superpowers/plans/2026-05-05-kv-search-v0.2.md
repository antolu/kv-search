# kv-search v0.2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `KeywordQuery` dataclass and `SearchHit.metadata` to kv-search, updating the `KeywordSearchBackend` ABC and engine coercion accordingly.

**Architecture:** `KeywordQuery` is a subclassable frozen dataclass wrapping `list[str]`; the engine coerces `list[str]` to `KeywordQuery` before passing to backends, so call sites don't change. `SearchHit` gains a `metadata: dict[str, Any]` field defaulting to an empty dict.

**Tech Stack:** Python 3.11+, dataclasses, pytest, pre-commit (ruff + mypy)

Find the `keyword_search` method in `_engine.py`. Change the signature to accept `KeywordQuery | list[str]` and add coercion:

```python
async def keyword_search(
    self,
    session: SearchSession,
    queries: kv_search._types.KeywordQuery | list[str],
) -> list[SearchHit]:
    if self._keyword_backend is None:
        raise RuntimeError("No keyword search backend configured")
    if isinstance(queries, list):
        queries = kv_search._types.KeywordQuery(queries=queries)
    hits = await self._keyword_backend.keyword_search(queries)
    session.add_keyword_hits(hits)
    return hits
```
